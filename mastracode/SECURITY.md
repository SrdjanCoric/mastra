# Security policy

## Reporting a vulnerability

Do not post credentials, bot tokens, private Telegram identifiers, message content, or exploit details in a public issue.

Report vulnerabilities through GitHub's private vulnerability reporting for `SrdjanCoric/mastra`. Include the affected version, impact, reproduction steps, and any suggested mitigation. If private reporting is unavailable, open a public issue that asks for a private contact channel without including vulnerability details.

## Supported versions

Security fixes are applied to the latest published version of `mastracode-remote`. Upgrade to the latest release before reporting an issue that may already be fixed.

## Scope

Reports are especially useful when they involve Telegram authorization, prompt or approval routing, secret storage, local IPC, project isolation, package installation, or unintended changes to legacy `mastracode-remote` state.

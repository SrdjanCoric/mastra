## 9. Expected Repository Files

A mature repository will commonly contain equivalents of the following files. Exact paths may differ by hosting platform or ecosystem.

- [ ] `README.md`
- [ ] `CONTRIBUTING.md`
- [ ] `LICENSE`
- [ ] `SECURITY.md`
- [ ] `CODEOWNERS`
- [ ] `.editorconfig`
- [ ] `.env.example` or equivalent
- [ ] A dependency lock file
- [ ] A runtime or toolchain version file
- [ ] Linter configuration
- [ ] Formatter configuration
- [ ] Strict type-checker configuration where applicable
- [ ] Test and coverage configuration
- [ ] Pre-commit hook configuration
- [ ] CI workflow configuration
- [ ] Dependency update automation configuration
- [ ] Architecture documentation
- [ ] Architecture Decision Records or equivalent
- [ ] API or public-interface documentation where applicable
- [ ] Repository-specific AI coding instructions
- [ ] Container or reproducible development-environment configuration where useful

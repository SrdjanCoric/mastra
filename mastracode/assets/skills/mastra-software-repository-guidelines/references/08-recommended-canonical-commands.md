## 8. Recommended Canonical Commands

The exact names may follow ecosystem conventions, but the repository should expose clear equivalents for the following operations:

- [ ] `setup` — install dependencies and initialize the local environment.
- [ ] `dev` — start the development workflow.
- [ ] `format` — apply automatic formatting.
- [ ] `format-check` — verify formatting without changing files.
- [ ] `lint` — run linting and static analysis.
- [ ] `typecheck` — run strict type checking where applicable.
- [ ] `test` — run the default automated test suite.
- [ ] `test-coverage` — run tests with coverage reporting.
- [ ] `test-integration` — run integration or end-to-end tests where applicable.
- [ ] `build` — produce the project's distributable or deployable output.
- [ ] `check` — run the complete local validation suite expected before a pull request.

The README and contributing guide should identify the canonical command names used by the repository.
